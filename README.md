# 星韵麻友 (magmaj) - 日式麻将在线游戏

一款基于现代 Web 技术栈开发的日式麻将（日麻）在线游戏，提供完整的日麻规则体验、实时多人对战、可视化牌桌界面。

---

## 项目简介

**星韵麻友** 是一款面向女性向轻竞技玩家的日式麻将游戏，采用现代化的前端技术栈，实现完整的日麻规则系统、实时多人对战、精美的可视化界面。游戏参考雀魂的界面风格，主界面设计参考金铲铲的布局，提供统一的游戏体验。

### 核心特色

- 🎴 **完整日麻规则** - 支持吃碰杠、立直、自摸、荣牌、流局等完整日麻流程
- 🏆 **役种判定系统** - 覆盖断幺九、清一色、国士无双等主要役种，自动计算番数符数
- 💰 **专业分数结算** - 严格遵循日麻分数计算公式，支持满贯/跳满/倍满/役满判定
- 🌐 **实时多人对战** - 基于 WebSocket 和 Colyseus 实现多人在线匹配和对局同步
- 🎨 **精美可视化界面** - 使用 PixiJS 实现流畅的牌桌动画和交互
- 📱 **多端适配** - 支持桌面端和移动端访问，响应式布局设计

## 核心功能模块

### 1. 麻将规则引擎 (`src/core/mahjong/`)

#### 牌型检测与役种判定 (`evaluator.ts`)

- **标准牌型检测** - 4副面子（刻子/顺子）+ 1对将牌
- **七对子检测** - 7个对子的特殊牌型
- **国士无双检测** - 13种幺九牌齐备的役满牌型
- **役种自动判定** - 断幺九、清一色、混一色、对对和、役牌、门清自摸、混老头、三杠子等
- **听牌分析** - 13张手牌时自动计算可胡牌的牌型

#### 分数计算系统 (`scoring.ts`)

- **基础点数公式** - `基础点数 = 符数 × 2^(番数+2)`
- **满贯判定** - 5番以上或基础点数≥2000时触发满贯
- **级别划分** - 满贯(2000) → 跳满(3000) → 倍满(4000) → 三倍满(6000) → 役满(8000)
- **自摸分配** - 庄家自摸：闲家各付2倍；闲家自摸：庄家付2倍，其他闲家各付1倍
- **荣牌分配** - 点炮者单独支付（庄家点炮付6倍基础分，闲家点炮付4倍）

#### 对局流程管理 (`flow.ts`)

- **牌墙创建** - 生成136张完整牌组并随机洗牌
- **发牌机制** - 庄家14张，闲家各13张，14张王牌留作岭上开花
- **吃碰杠检测** - 自动判断是否可以吃/碰/杠
- **玩家顺序** - 按顺时针顺序管理出牌权转移

### 2. 游戏状态管理 (`src/store/`)

#### 对局状态 (`game.ts`)

- 玩家列表及手牌、弃牌、副露管理
- 当前回合状态（摸牌/出牌/等待）
- 最近动作记录和实时手牌分析
- 胡牌结算和分数分配逻辑

#### 玩家认证 (`auth.ts`)

- 玩家登录状态管理
- 星核余额管理（游戏内货币）
- 可扩展为对接真实用户系统

#### 匹配与房间 (`match.ts`, `colyseus.ts`)

- 在线玩家列表管理
- 房间创建与加入
- 游戏开始同步
- 实时聊天消息

### 3. 匹配服务器 (`server.ts`)

基于 Bun 原生 WebSocket 实现的轻量级匹配服务：

- 维护客户端连接状态
- 处理匹配开始/停止请求
- 广播房间状态更新
- 支持动态玩家名称分配

### 4. 前端页面系统

#### 登录页面 (`Login.vue`)

- 支持微信、QQ、邮箱三种登录方式
- 简洁美观的登录界面
- 游客模式快速体验

#### 主大厅 (`Home.vue`)

- 玩家头像和昵称展示
- 星核余额显示
- 开始游戏按钮
- 充值中心入口
- 个人设置面板

#### 游戏对局 (`GamePlay.vue`)

- PixiJS 渲染的牌桌界面
- 实时摸牌/弃牌/吃碰杠操作
- 手牌分析面板（番数、符数、役种）
- 分数结算展示
- 玩家信息和对局状态

## 日麻规则概览

### 基本概念

- **牌型** - 共34种牌：万(1-9)、筒(1-9)、索(1-9)、字牌(东南西北中发白)，每种4张
- **胡牌条件** - 4副面子（刻子3张相同 / 顺子3张连续） + 1对将牌，且至少1番
- **特殊牌型** - 七对子（7个对子）、国士无双（13种幺九牌齐备）

### 主要役种与番数

| 役种 | 番数 | 说明 |
|------|------|------|
| 断幺九 | 1番 | 手牌不含幺九牌（1/9万筒索及字牌） |
| 役牌 | 1番 | 包含中/发/白或自风/场风的刻子 |
| 门清自摸 | 1番 | 无副露且自摸胡牌 |
| 对对和 | 2番 | 4个刻子（不含顺子） |
| 混一色 | 3番 | 仅含一种花色牌+字牌 |
| 三杠子 | 2番 | 包含3个杠子 |
| 混老头 | 2番 | 仅含幺九牌 |
| 清一色 | 6番 | 仅含一种花色牌（不含字牌） |
| 国士无双 | 役满 | 13种幺九牌齐备 |

### 分数计算示例

- **30符4番** - 基础分 = 30 × 2^(4+2) = 1920 → 未满贯
- **40符3番** - 基础分 = 40 × 2^(3+2) = 1280 → 未满贯
- **5番** - 直接满贯 = 2000点基础分
- **7番** - 跳满 = 3000点基础分
- **10番** - 倍满 = 4000点基础分
- **13番以上** - 役满 = 8000点基础分

## 快速开始

### 环境要求

- **Bun** ≥ 1.0 (推荐) 或 **Node.js** ≥ 18
- 现代浏览器（Chrome/Edge/Firefox/Safari）

### 安装依赖

```bash
# 使用 Bun（推荐）
bun install

# 或使用 npm
npm install
```

### 启动开发服务器

```bash
# 启动前端开发服务器
bun run dev

# 启动匹配服务器（另开终端）
bun run serve
```

### 构建生产版本

```bash
bun run build
```

### 运行测试

```bash
bun test src
```

## 核心代码示例

### 手牌分析

```typescript
import { analyzeHand } from './src/core/mahjong'

// 分析14张手牌
const result = analyzeHand(tiles, openMelds, {
  isMenzen: true,      // 是否门清
  isTsumo: true,       // 是否自摸
  seatWind: 'east',    // 自风
  roundWind: 'east'    // 场风
})

console.log('是否胡牌:', result.isWinning)
console.log('番数:', result.han)
console.log('符数:', result.fu)
console.log('役种:', result.yaku.join('、'))
console.log('听牌:', result.possibleWinningTiles)
```

### 分数计算

```typescript
import { calculatePayments, formatLimit } from './src/core/mahjong/scoring'

// 计算自摸胡牌的分数分配
const payment = calculatePayments(han, fu, {
  isTsumo: true,       // 自摸
  isDealer: false,     // 是否庄家
  playerCount: 4       // 玩家人数
})

console.log('满贯级别:', formatLimit(payment.limit))
console.log('赢家获得:', payment.winnerGain)
console.log('各玩家支付:', payment.payments)
```

## 开发指南

### 添加新役种

1. 在 `src/core/mahjong/evaluator.ts` 的 `detectYaku` 函数中添加检测逻辑
2. 在 `analyzeHand` 函数的 `yakuHanMap` 中添加番数映射
3. 更新役种列表和文档

### 添加新页面

1. 在 `src/pages/` 创建新的 Vue 组件
2. 在 `src/router/index.ts` 中注册路由
3. 在主界面添加导航入口

### 状态管理规范

- 使用 Pinia 管理全局状态
- 按功能模块划分 Store（game/auth/match/colyseus）
